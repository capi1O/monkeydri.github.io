Virtual Machine Compatibility

Posted on 18 May 2018 2 mins read

https://www.eevblog.com/forum/altium/will-altium-17-run-in-virtualbox-or-vmware-on-a-linux-host/

solidworks PCB keeps opening new instance rather than open PCBdoc inside : https://forum.solidworks.com/thread/28754#128290

Vmware 3D acceleration support for intel HD graphics on ubuntu

needs to add line to either ~/.vmware/preferences.ini or to the vmx fie itself :

mks.gl.allowBlacklistedDrivers = "TRUE" source : https://ubuntuforums.org/showthread.php?t=2368057 & https://askubuntu.com/questions/537787/enable-3d-hw-acceleration-on-vmware-workstation-10-on-ubuntu-14-04

altium 18 on virtualbox requires DirectX 10 :

This version of Altium Designer’s PCB editor has been optimized for use with DirectX 10, and requires a graphics card that supports this

source : https://www.altium.com/documentation/18.0/display/ADES/Altium+Designer+-+((System+Requirements))

run > dxdiag > display tab :

main driver : VBoxDispD3D.dll v 5.2.12.0 feature levels 9_3 9_2 9_1

source : https://community.esri.com/thread/187265-how-to-check-directx-feature-level

DirectDraw Acceleration : Enabled Direct3D Acceleration : Enabled AGP Texture Acceleration : Not Available

same dxdiag in vmware (working altium 18) :

feature levels 10_0 9_3 9_2 9_1 DirectDraw Acceleration : Enabled Direct3D Acceleration : Enabled AGP Texture Acceleration : Enabled

Virtualbox does not support DX10 features :

it appears that the 2D and 3D options in the [VirtualBox] settings enable genuine hardware acceleration for pre-10 versions of DirectX and without those options a DX8 or DX9 application gets told that there is no decent hardware. By contrast, a DX10 or DX11 application is always promised hardware support [DX11 “declared”] but never gets it.

source : https://forums.virtualbox.org/viewtopic.php?f=2&t=74964
