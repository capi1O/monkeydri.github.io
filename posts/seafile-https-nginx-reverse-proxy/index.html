Seafile HTTPS nginx reverse-proxy

Posted on 18 May 2018 2 mins read

config for root domain ie. seafile.domain.com

set SEAFILE_URL to https://seafile.domain.com in docker-compose.yml.

location /seafile {
	proxy_pass					http://seafile;
	proxy_set_header		Host $host;
	proxy_set_header		Forwarded "for=$remote_addr;proto=$scheme";
	proxy_set_header		X-Forwarded-For $remote_addr;
	proxy_set_header		X-Forwarded-Proto $scheme;
	proxy_set_header		X-Real-IP $remote_addr;
	proxy_set_header		Connection "";
	proxy_http_version	1.1;
	proxy_set_header		X-Forwarded-Proto https;
	proxy_read_timeout  1200s;

	access_log /var/log/nginx/seafile-access.log;
	error_log /var/log/nginx/seafile-error.log;
}

source : https://forum.seafile.com/t/solved-using-nginx-reverse-proxy-for-seafile-docker/6109/7 & https://manual.seafile.com/deploy/https_with_nginx.html

config for non root domain ie. domain.com/seafile

set SEAFILE_URL to https://domain.com/seafile in docker-compose.yml.

In additions to the /seafile block in main nginx config, we need to modify settings_seahub.py, but after setup.

launch & setup seafile docker dcu -d, get MySQL root random password from logs docker-compose seafile logs and dce seafile setup

copy the contents of settings_seahub.py : dce seafile bash, cat /seafile/conf/seahub_settings.py

create a file settings_seahub.py on host with its content and additional settings :

SERVE_STATIC = False
MEDIA_URL = '/seafile/seafmedia/'
COMPRESS_URL = MEDIA_URL
STATIC_URL = MEDIA_URL + 'assets/'
SITE_ROOT = '/seafile/'
LOGIN_URL = '/seafile/accounts/login/'

mount it add - /var/docker/seafile-docker-foxel/seahub_settings.py:/seafile/conf/seahub_settings.py to volumes sections of docker-compose.yml

then modify seafile internal nginx config

copy the contents of seafile internal nginx config seafile.conf : dce seafile bash, cat /etc/nginx/sites-available/seafile.conf

create a file nginx-seafile.conf on host with its content and modify it :

enable HTTPS in main block fast_cgi :

location / {
		...
		fastcgi_param   HTTPS               on;
		fastcgi_param   HTTP_SCHEME         https;
		...
}

replace media block :

location /seafile/seafmedia {
		rewrite ^/seafile/seafmedia(.*)$ /media$1 break;
		root /opt/seafile/latest/seahub;
}

mount it add - /var/docker/seafile-docker-foxel/nginx-seafile.conf:/etc/nginx/sites-available/seafile.conf to volumes sections of docker-compose.yml

source : https://manual.seafile.com/deploy/deploy_seahub_at_non-root_domain.html

relaunch docker dcs && dcu -d (dcr will not add new mounts)

bug : images are not displayed

When performing those operations on an existing seafile setup, images will not be displayed. This is because seafile’s cache.

ex avatar URL is https://domain.com/seafmedia/avatars/default.png instead of https://domain.com/seafile/seafmedia/avatars/default.png.

cache needs to be reset : dce seafile bash, rm -rf /tmp/seahub_cache/

source : https://github.com/haiwen/seahub/issues/1988
